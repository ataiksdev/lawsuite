# Deployment on Render: Environment Variables

This document lists the environment variables you need to configure in the Render dashboard for the LegalOps application.

## Backend Service (`legalops-backend`)

The following variables should be set in the Render dashboard for the `legalops-backend` service. Some are linked automatically in `render.yaml`.

| Variable | Required | Description |
|----------|----------|-------------|
| `JWT_SECRET` | Yes | Secret key for JWT signing. Can be auto-generated by Render. |
| `ENCRYPTION_KEY` | Yes | Fernet key for encrypting OAuth tokens at rest. Generate using: `python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"` |
| `APP_URL` | Yes | The full URL of your backend service (e.g., `https://legalops-backend.onrender.com`). |
| `FRONTEND_URL` | Yes | The full URL of your frontend service (e.g., `https://legalops-frontend.onrender.com`). |
| `GOOGLE_CLIENT_ID` | Optional | Google OAuth Client ID. |
| `GOOGLE_CLIENT_SECRET`| Optional | Google OAuth Client Secret. |
| `GOOGLE_REDIRECT_URI` | Optional | Google Workspace redirect URI: `https://<backend-url>/integrations/google/callback` |
| `GOOGLE_SIGNIN_REDIRECT_URI` | Optional | Google Sign-in redirect URI: `https://<backend-url>/auth/google/callback` |
| `PAYSTACK_SECRET_KEY` | Yes | Your Paystack Secret Key. |
| `PAYSTACK_PUBLIC_KEY` | Yes | Your Paystack Public Key. |
| `PAYSTACK_FREE_PLAN_CODE`| Optional | Paystack plan code for the Free plan. |
| `PAYSTACK_PRO_PLAN_CODE` | Optional | Paystack plan code for the Pro plan. |
| `PAYSTACK_AGENCY_PLAN_CODE`| Optional | Paystack plan code for the Agency plan. |
| `PLATFORM_ADMIN_ORG_ID`| Optional | Your Organisation ID to enable Admin features. Set this after registration. |

## Frontend Service (`legalops-frontend`)

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_API_URL` | Yes | The full URL of your backend service. (Linked automatically if using `render.yaml`) |
| `NEXT_PUBLIC_ENABLE_DEMO_MODE`| No | Set to `true` to enable demo mode features. |

## Notes

- `DATABASE_URL`, `DATABASE_URL_SYNC`, and `REDIS_URL` are automatically managed by Render and linked via the `render.yaml` blueprint.
- Ensure that `APP_URL` and `FRONTEND_URL` do **not** have a trailing slash.
- After the first deployment, you will need to run the seed script if you want demo data: `poetry run python -m scripts.seed` (can be done via Render's "Shell" tab in the backend service).
